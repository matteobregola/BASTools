Attack

Ref Feature Name Description
C1 APT The tool permits emulating a known Advanced Persistent   Threat’s (APT) attack path without manually configuring all the activities to   simulate it.
C2 Group of APTs The tool permits emulating multiple known Advanced Persistent   Threats’ (APTs) attack paths without manually configuring all the activities   to simulate them
C3 Lateral Movement The tool permits to perform lateral movement simulations [1].
C4 Multiple Targets • It permits to speed up the time required to perform the   simulations,
C5 Multi Objectives The activities span across different categories of attacks and   not only specific ones [1].
C6 Pre-Compromise The tool permits to execute activities belonging to the   pre-compromise phase[1][2]
C7 Reusable Information • The tool can collect data obtained through one activity and   reuse it in other activities (e.g., passwords, usernames…) [1][2].
C8 System Detection The tool is capable of automatically recognising if the   activities are supported by the targets, avoiding running incompatible   activities [1]
C9 Activities Filtering The tool allows filtering among possible activities to   determine which ones to include in the simulation [1][2][3].
C10 Activities Combined Filtering The tool allows filtering among possible activities, combining   different attributes, to determine which ones to include in the simulation
C11 Target Filtering The tool permits to choose the targets of an activity by fitlering on the target attributes
C12 Planners The tool permits to use or create planners that are capable of   deciding in which order and how to execute the activities [1][2]
C13 Prioritisation The tool permits running the activities chosen for the   simulation in a prioritized order [1]

The features in this section aim to analyse what are the general attack capabilities of the software. This list comprehend also last four features (C9 to C13) that are related more to the “configuration” of the attack, meaning the selection of targets, activities and their order. In particular the features “Activity Filtering” and “Activities Combined Filtering” are the attributes used to verify if the tool permits to filter the activities based on the elements described in the “Information and Filtering” subsection.
