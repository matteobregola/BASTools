Comparison Elements

Ref Feature Name Description
H1 Coverage The tool can be evaluated based on how many activities it   offers and how much these activities can cover different type of attacks [1].
H2 Effectiveness The tool can be evaluated based on how effective the   activities and simulations proposed are.
H3 Price and ROI The tool can be evaluated based on how much it costs, what are   the payment modalities and what is the predicted return on investment.
H4 Privacy The tool can be evaluated based on its ability to safeguard   user’s privacy.
H5 Security The tool can be evaluated based on it’s security level
H6 Customer Support The tool can be evaluated based on the customer support   offered by the company or community that developed the tool
H7 Integrability The tool can be evaluated based on its ability to integrate   with other solutions or, conversely, incorporate other software
H8 Operator Expertise The tool can be evaluated based on the level of cybersecurity   expertise request by the software users
H9 Resources Usage The tool can be evaluated based on how many resources consume   on the system that are target of the simulation
H10 Scalability The tool can be evaluated based on how many assets it can   support and how easy it is to scale this number
H11 Customization The tool can be evaluated based on how much it can be   customizable and adapted to align the simulation capabilities with the user   system and business interest
H12 Updated The tool can be evaluated based on how much it’s updated, both   from an attacker's perspective and from the technology in general
H13 Sources of CTI The tool can be evaluated based on how vast and valuable is   the knowledge behind the tool[1].
H14 Tool Improvement The tool can be evaluated based on how feasible it is to   improve the tool's capabilities by developing new features or creating   plugins[1
H15 Time Required How much time the tool requires to perform an activity
H16 Influence The tool can be evaluated based on how much their actions   influence the system's result[1][2].
H17 Mitigations Effectivness The tool can be evaluated based on how much support it offers   for fixing the vulnerabilities [1].

As explained in 'Framework Design', this final section of the 'Tool Analysis Framework' aims to enumerate the elements that can be used to directly compare two solutions. Unlike the previous features, these elements may be difficult to evaluate objectively, and developing a precise metric can be challenging in some cases. Finally, as noted in the introduction of this chapter, it is important to remember that this section focuses on highlighting the features that are valuable to consider when comparing two BAS software, omitting the general comparison elements applicable to software in general.
