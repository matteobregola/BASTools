Results

Ref Feature Name Description
D1 Manual Report The tool permits the user to generate a report after the   simulation[1]
D2 Automatic Report The tool automatically generates one or more reports after the   simulation
D3 Automatic Report Sending The tool can be configured to automatically generate reports   and send them to specific users thanks to mail, ticketing systems or other   integrations
D4 Role based Reports The tool generates reports with different styles based on the   receivers’ role [1].
D5 Analytics The tool returns the analytics of the simulation with a   parsable format or offers an API to do use them[1][2].
D6 Results Filtering The tool’s GUI permits filtering the results [1].
D7 Framework-Based Results The presents the results mapping them to a specific   framework’s mapping [1].
D8 System Response The report categorises in different levels how the system   responded to the various activities of the simulation[1
D9 System Vulnerability Levels The tool offers a quantitative evaluation of how much the   system is vulnerable to each activity performed during the simulation[1][2][3
D10 Targets Vulnerability Levels The tool offers a quantitative evaluation of how much each   target is vulnerable to each activity performed during the   simulation[1].
D11 System Overall Vulnerability Level The tool offers a quantitative evaluation of how much the   system is vulnerable based on the simulation result[1][2][3].
D12 Targets Overall Vulnerability Level The tool offers a quantitative evaluation of how much each   target is vulnerable based on the simulation result[1][2
D13 System Risk Levels The tool delivers a quantitative evaluation of the risk level   to the system resulting from the activities simulated [1][2][3]
D14 Targets Risk Levels The tool offers a quantitative evaluation of how high the risk   for each target resulting from the activities simulated [1
D15 System Overall Risk Level The tool offers a quantitative evaluation of how high the risk   is for the system based on the simulation result[1][2][3].
D16 Targets Overall Risk Level The tool offers a quantitative evaluation of how high the risk   is for each target based on the simulation result[1][2]
D17 Vulnerabilities Visualization The tool permits to graphically visualize the vulnerabilities   of the system in a meaningful way [1][2].
D18 Threat Grouping The tool groups the threats discovered during the simulation   in a specific way
D19 Attack Path Visualization The tool enables graphical visualization of the attack path of   the simulation, displaying the sequence of steps performed [1].
D20 Discovered Targets If the simulation involved lateral movement, the tool provides   the list of discovered targets from the initial breach point in which the   attack has started
D21 Compromised Targets If the simulation involved lateral movement, the tool provides   the list of discovered targets that permitted the simulation to run   activities on their systems
D22 Lateral Movement Visualization If the simulation involved lateral movement, the tool provides   a visual representation on how the attacker performed it.
D23 Benchmark The report provides a benchmark of the customer readiness in   comparison with the others competitors within the same industry[1]
D24 Mitigation insights The result contains suggestion on how to fix the discovered   vulnerabilities and security problems [1]
D25 Comparison The tool permits to perform a comparison of the simulation's   result with previous ones
D26 Integration Comparison The tool permits to perform a comparison of the simulation's   result based on the system’s security integrations[1
D27 Live Results The tool is capable of generating results and reports during   the simulation execution
D28 Outputs The tool is capable of collecting the outputs of the   activities’ commands and adding them as part of the results
D29 Logs The tool is capable of collecting the system logs related to   the simulation’s activities

The “Results” section comprehends the features describing which type of results regarding the simulations are presented and how. An effective tool grants a clear understanding to the user about how the system responded to the simulation, which targets are more vulnerable and which activities represent the highest risk for the system. It is important to provide the user with as much information as possible but also to permit him to filter or visualize only the ones that he considers useful[1]. Another fundamental aspect of BAS that is analysed in this section is providing the user with suggestions on how to fix its vulnerabilities.
