Organization

Ref Feature Name Description
B1 Ability The tool offers a list of pre-defined abilities that can be   run on the target system
B2 Ability Configuration The tool permits personalizing an ability by changing   pre-defined parameter
B3 Custom Ability The tool permits to create and run on-fly an ability, this   ability can be built from scratch or by modifying an existing one
B4 Custom Ability Saving The tool permits to save permanently and reuse a custom   ability
B5 Import Ability The tool permits importing an ability from a file
B6 Operation The tool can run a set of abilities in order to perform an   operation on one or multiple target systems [1].
B7 Operation Configuration The tool permits personalizing an Operation by changing   pre-defined parameters or performing other operations [1].
B8 Custom Operation The tool permits to create and run on the fly an operation by   pipelining different abilities or modifying an existing one
B9 Custom Operation Saving The tool permits to save permanently and reuse a custom   operation.
B10 Import Operation The tool permits importing an operation from a file

The Breach and Attack Simulation tools that are present on the market offer different simulation capabilities that can be distinguished not only by their objectives but also by h ow they are structured. Many solutions subdivide the general concept of simulation into smaller elements that can be grouped or chained to perform a simulation. There are different possibilities for performing this subdivision, this framework provides a two-element subdivision that is considered to be the most suitable option for a comparison with the other tools [1]. These elements are named “Ability” and “Operation”. An “Ability” has a basic objective achievable by running a few commands, multiple abilities can be grouped into an “Operation” to reach a more complex goal [2]. A simulation involves one or many operations. Therefore the features of this section permit the reader to verify if the tool offers a logical structure similar to this, how much is it customisable and what are the benefits of having multiple fine-grain elements rather than a whole “simulation” object.
